Dokku

(Docker + Heroku)

Deploy apps to the cloud using 'GIT PUSH' on AWS, Google Cloud, Azure, Linode, DigitalOcean

A Platform as a Service (PaaS) that
makes app deployment fast and simple.


git push heroku master

Uses Git to upload your app and automatically configure the server using a buildpack.

What are buildpacks?

Buildpacks = installation scripts

Depending on the programming language, the scripts will retrieve dependencies, output generated assets or compiled code, and more.

Heroku currently has buildpacks for:
Ruby, Python, Java, Clojure,
Node.js, Scala, Go and PHP

All of Heroku's buildpacks are open-source.

What is Dokku & why should I use it?

Heroku is expen$ive!

Dokku is a cheap, Docker powered mini-Heroku.

Cost comparison here...

tl;dr => Heroku PaaS inside a Docker container. Fast, easy deployments of apps and static sites. Able to be run on cheap micro-servers on nearly any cloud provider

Open-source project released in 2013 by Jeff Lindsay. Over 13,000 stars 260+ contributors on Github.

Multiple sites / apps / micro-services on one domain.

Not tied to any one cloud hosting provider. Can be deployed in many...

Currently supported on:

  • Ubuntu 16.04 x64
  • Ubuntu 14.04 x64
  • Debian 8.2 x64
  • CentOS and Arch Linux (experimental)

Uses actual Heroku buildpacks (open sourced by Heroku) or CUSTOM buildpacks.

Plugins are available for quick installation and configuration of popular tools. Give current list of plugins. We will use the plugin Dokku-Letsencrypt to secure our website (HTTPS).

Caveats

  • Only as secure as an open source project can be
  • Best for: small deployments, internal apps, testing
  • (Bug) Must re-push apps after server reboot (Docker changed the way networking works... explain this...)
